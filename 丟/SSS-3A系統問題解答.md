# SSS-3A 系統問題解答

## 一、查詢條件相關問題

### 3. 實際的查詢日期是看那一個？是期間變化日期區間還是期初申購日大於？

**答案：** 實際的查詢日期主要使用**期間變化日期區間**（起日和訖日），但同時也會檢查**期初申購日大於**（init_date）的條件。
所有契約的申購日必須 >= init_date。預設值為 2008/01/01，這是為了排除過早的歷史資料，只統計從 init_date 之後的契約。
**說明：**
- **有效契約金額、累計扣款金額、截止日餘額**等存量指標：使用「申購日 <= 訖日 且 到期日 > 訖日」的條件，統計截至訖日的有效契約。
同時也會檢查「申購日 >= init_date」的條件。

### 4. 依客戶、依契約差在那裡？

**答案：** 

**依客戶：**
- 系統會先計算每個客戶在所有有效契約中的總金額
- 再篩選總金額符合 min_amount 和 max_amount 範圍的客戶
- 只要客戶的總金額符合範圍，該客戶的所有契約都會被納入統計

**範例說明：**
- 客戶A有三個契約，金額分別是 10萬、20萬、30萬，總計 60萬
- 如果設定金額範圍是 50萬到 100萬，則客戶A的三個契約都會被納入

**依契約：**
- 直接篩選每個契約的原始金額是否符合 min_amount 和 max_amount 範圍
- 每個契約獨立判斷，只有金額符合範圍的契約才會被納入統計

**範例說明：**
- 客戶A有三個契約，金額分別是 10萬、20萬、30萬
- 如果設定金額範圍是 15萬到 25萬，則只有 20萬的契約會被納入

## 二、明細相關問題

### 2. 契約淨額是指什麼？

**答案：** 契約淨額 = **新增契約金額 - 到期契約金額**

**說明：**
- **新增契約金額**：在期間變化日期區間內（起日到訖日）新增的基金契約金額總和
- **到期契約金額**：在期間變化日期區間內（起日到訖日）到期的基金契約金額總和
- **契約淨額**：兩者相減，表示期間內契約金額的淨變化。如果為正數，表示契約金額增加；如果為負數，表示契約金額減少。


### 3. 停利金額是指已達停利該子基金的買回總金額是嗎？

**答案：** 這邊停利金額是指 效率投資（子基金獲利了結）的所有申購單總金額

### 4. 扣款金額和累計扣款金額差在那裡？扣款金額是用期間變化的日期區間統計是嗎？而累計扣款金額是看有效契約中不管日期已累扣了多少金額？

**答案：** 
**扣款金額：**
- 這是**期間變化**的指標，只統計期間內新增的扣款金額
- 日期條件：只統計在起日到訖日之間發生的扣款交易
- 例如：客戶設定定期定額扣款，系統會在每次扣款時產生一筆扣款交易記錄

**累計扣款金額：**
- 這是**存量**指標，統計有效契約中截至訖日的累計扣款金額
- 不管扣款是在什麼時候發生的，只要契約在訖日時仍然有效，就會累計該契約的所有扣款金額

**範例說明：**
- 假設起日 = 2024/01/01，訖日 = 2024/12/31
- 契約A：2023/06/01 申購基金，2025/06/01 到期
  - 2023/07/01 扣款 10萬
  - 2024/02/01 扣款 10萬
  - 2024/08/01 扣款 20萬
- 契約B：2024/03/01 申購基金，2025/03/01 到期
  - 2024/05/01 扣款 15萬
  - 2024/11/01 扣款 25萬

**扣款金額**（期間變化）：只統計 2024/01/01 到 2024/12/31 之間發生的扣款
- 契約A：2024/02/01 扣款 10萬 + 2024/08/01 扣款 20萬 = 30萬
- 契約B：2024/05/01 扣款 15萬 + 2024/11/01 扣款 25萬 = 40萬
- 總計 = 30萬 + 40萬 = 70萬

**累計扣款金額**（有效契約存量）：統計截至 2024/12/31 仍有效的契約的所有扣款
- 契約A 在 2024/12/31 仍有效，累計所有扣款 = 10萬 + 10萬 + 20萬 = 40萬
- 契約B 在 2024/12/31 仍有效，累計所有扣款 = 15萬 + 25萬 = 40萬
- 總計 = 40萬 + 40萬 = 80萬

### 5. 有效契約金額是指此母基金截至輸入的迄日的市值？成本？

**答案：** 有效契約金額是指效率投資（期初母基金申購）的金額,是**成本**，不是市值。

**說明：**
- 這裡的「金額」是契約的原始申購金額＊匯率，不是根據淨值計算的市值
- 日期條件：契約申購日 <= 訖日 且 契約到期日 > 訖日（表示契約仍有效）


### 6. 截止日餘額是指母基金嗎？

**答案：** 是的。截止日餘額（FUM）是指母基金截至訖日的**市值**。

**說明：**
- 計算公式：憑證數 × 淨值 × 匯率 = 台幣市值
- 日期條件：使用訖日當日的淨值
- 只統計在訖日時仍有效的憑證證明（已發行且未註銷）
- 這裡的「餘額」是根據淨值計算的市值



### 7. 全部契約金額是指有效＋無效？

**答案：** 是的。全部契約金額是指截至訖日的所有契約金額，包括有效和無效的契約。

**說明：**
- 不管契約是否已經到期或終止，只要申購日在訖日之前，就會被納入統計
- 這與「有效契約金額」的差別在於：有效契約金額只統計在訖日時仍然有效的契約，而全部契約金額統計所有契約（不管是否有效）

**範例說明：**
- 假設訖日 = 2024/12/31
- 契約A：2023/01/15 申購，2024/06/01 到期（已到期）→ 會被納入「全部契約金額」，但不會被納入「有效契約金額」
- 契約B：2024/03/01 申購，2025/03/01 到期（仍有效）→ 會被納入「全部契約金額」和「有效契約金額」

### 8. 手續費在停利金額右側的是指子轉母手續費是嗎？而全部契約的手續費是指母轉子？

**答案：** 
手續費是指以下交易類別產生的（申購檔金額＊匯率）總和
L2 效率投資 (期初母基金申購) OR L3 效率投資 (母基金轉投資子基金) OR L4 效率投資 (子基金獲利了結)


手續費 跟 全部契約的手續費邏輯大致一樣 只是抓取的時間點不一樣 條件如下
手續費： 
範圍 :   設定起日 ＝< 日期 ＝< 設定迄日 且 期初申購日大於 <= 日期 

全部契約的手續費
範圍 :    日期 ＝< 設定迄日 且 期初申購日大於 <= 日期 



### 9. 以上的金額在換算為台幣時，皆是按契日的匯率進行換算嗎？

**答案：** 是的。所有金額在換算為台幣時，都是按**契約日**的匯率進行換算。

**說明：**
- 每個契約都使用其申購日當日的匯率來換算，確保歷史資料的一致性
- 即使查詢日期不同，匯率已經變化，系統仍然使用契約日的匯率來換算

**範例說明：**
- 契約A：2024/01/15 申購，金額 10,000 美元，2024/01/15 的美元匯率 = 30 → 台幣金額 = 10,000 × 30 = 300,000 元
- 契約B：2024/06/01 申購，金額 10,000 美元，2024/06/01 的美元匯率 = 31 → 台幣金額 = 10,000 × 31 = 310,000 元
- 即使查詢日期是 2024/12/31，匯率已經變成 32，系統仍然使用各自的契約日匯率來換算：
  - 契約A 仍然使用 30（2024/01/15 的匯率）
  - 契約B 仍然使用 31（2024/06/01 的匯率）

### 10. 下方契約內的類別有那些？另新戶怎麼定義？

**答案：** 

**類別：**
- **提前解約**：契約提前終止
- **餘額不足**：因餘額不足而終止契約
- **契約到期**：契約正常到期
- **空值**：契約仍有效，系統會根據「期初金額」判斷是否為新戶

**新戶定義：**
- **新戶**：如果客戶的「期初金額」**不在** min_amount 和 max_amount 之間，則顯示為「新戶」
- **舊戶**：如果客戶的「期初金額」**在** min_amount 和 max_amount 之間，則顯示為空（表示舊戶）

**期初金額的計算邏輯：**
- 統計該客戶在起日之前的所有有效母基金契約金額總和
- 條件：申購日 >= init_date 且申購日 <= (起日 - 1) 且到期日 > (起日 - 1)
- 金額會按契約日的匯率換算為台幣後加總

**範例說明：**
- 假設起日 = 2024/01/01，min_amount = 100萬，max_amount = 500萬
- 客戶A：在 2023/12/31 之前有有效契約，總金額 80萬（不在 100萬到 500萬之間）→ 顯示「新戶」
- 客戶B：在 2023/12/31 之前有有效契約，總金額 200萬（在 100萬到 500萬之間）→ 顯示空（舊戶）
- 客戶C：在 2023/12/31 之前沒有任何有效契約 → 顯示「新戶」

### 11. 金額級距指什麼

**答案：** 金額級距是指**契約金額本身**，用於動態查詢功能。

**說明：**
- 這是一個虛擬欄位，實際值就是契約的原始金額（原幣），不是換算後的台幣金額
- 主要用於系統的動態查詢功能，讓使用者可以根據金額級距進行篩選或分類
- 例如：可以設定金額級距為 0-10萬、10-50萬、50-100萬等，用來分類統計不同金額級距的契約

**範例說明：**
- 契約金額是 10,000 美元，金額級距就是 10,000（美元），不是換算後的台幣金額
