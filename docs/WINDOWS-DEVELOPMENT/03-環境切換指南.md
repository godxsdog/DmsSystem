# 環境切換指南

> 📋 說明如何在 Mac 開發環境和 Windows 開發環境之間切換

## 🔄 從 Mac 切換到 Windows

### 步驟 1：提交 Mac 的變更

**在 Mac 上：**
```bash
# 查看變更
git status

# 加入所有變更
git add -A

# 提交（使用規範的 commit 訊息）
git commit -m "feat: 描述你的變更"

# 推送到遠端
git push origin feature/your-branch-name
```

### 步驟 2：在 Windows 取得最新程式碼

**在 Windows 上：**
```powershell
# 如果還沒有 clone
git clone <repository-url>
cd DmsSystem

# 如果已經有程式碼
git pull origin main
# 或
git pull origin feature/your-branch-name
```

### 步驟 3：還原套件

```powershell
dotnet restore
```

### 步驟 4：設定 Windows 環境

1. **設定資料庫連接**（使用正式環境）
2. **不要使用 Mac 的測試資料腳本**
3. **參考 Windows 開發手冊**

## 🔄 從 Windows 切換到 Mac

### 步驟 1：提交 Windows 的變更

**在 Windows 上：**
```powershell
git add -A
git commit -m "feat: 描述變更"
git push origin feature/your-branch-name
```

### 步驟 2：在 Mac 取得最新程式碼

**在 Mac 上：**
```bash
git pull origin main
# 或
git pull origin feature/your-branch-name
```

### 步驟 3：設定 Mac 環境

1. **啟動 Docker SQL Server**
2. **執行測試資料腳本**（僅 Mac 使用）
3. **參考 Mac 開發手冊**

## ⚠️ 重要差異

### Mac 環境（個人開發）

- ✅ 使用 Docker SQL Server
- ✅ 使用測試資料腳本
- ✅ 使用 `appsettings.Development.json`
- ✅ 可以自由測試和實驗

### Windows 環境（團隊開發）

- ✅ 使用正式 SQL Server
- ❌ **不要**使用測試資料腳本
- ✅ 使用 `appsettings.Production.json`
- ✅ 遵循團隊開發規範
