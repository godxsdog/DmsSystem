# Windows 環境版本控制與更新指南

## 🔄 取得最新程式碼

### 從 Git 倉庫拉取最新程式碼

```powershell
# 開啟 PowerShell
cd C:\Your\Project\Path\DmsSystem

# 拉取最新程式碼
git pull origin main
```

### 還原 NuGet 套件

**在 Visual Studio 2022：**
1. 開啟 `DMS.sln`
2. 右鍵點擊方案 → 「還原 NuGet 套件」

**或使用命令列：**
```powershell
dotnet restore
```

### 重新建置專案

```powershell
dotnet clean
dotnet build
```

## 🔀 切換分支

### 查看所有分支

```powershell
git branch -a
```

### 切換到特定分支

```powershell
git checkout <branch-name>
```

### 建立新分支

```powershell
git checkout -b <new-branch-name>
```

## 📤 提交變更

### 查看變更狀態

```powershell
git status
```

### 加入變更

```powershell
# 加入所有變更
git add -A

# 或加入特定檔案
git add <file-path>
```

### 提交變更

```powershell
git commit -m "feat: 描述變更內容"
```

### 推送到遠端

```powershell
git push origin <branch-name>
```

## ⚠️ 重要提醒

1. **提交前檢查**：確認資料庫連接字串不包含敏感資訊
2. **環境變數**：不要將生產環境密碼提交到 Git
3. **測試**：提交前請先測試程式碼可以正常執行
4. **分支策略**：遵循團隊的分支策略和命名規範
